# 特化型深層学習モデル
# 株式取引最適化システム 設計計画書
# Deep Learning Model for Stock Trading Optimization

Version 1.0　|　2025年4月

 
## 1. プロジェクト概要
本計画書は、東京証券取引所を含む全銘柄を対象に、資産を最大化する取引判断を行う特化型深層学習モデルの設計・開発計画をまとめたものです。

### 1.1　目的
全銘柄を対象として、○ヶ月（目標期間）の中で最も効率よく資産を増やすための取引手段を、深層学習モデルが自動的に判断・推奨するシステムを構築する。

### 1.2　取引手段の定義（4分類）
モデルは以下4種の取引手段から、銘柄・期間ごとに最適なものを選択して出力する。

記号	手段名	概要	戦略分類
①	買って売る	スイング・モメンタム戦略	短〜中期トレード
②	買って持ち続ける	トレンドフォロー戦略	中〜長期ホールド
③	売って持ち続ける	空売りホールド戦略	下落局面活用
④	売って買う	ショートカバー戦略	反転局面活用

### 1.3　期間パラメータ設計方針
「○ヶ月で資産を増やす目標期間」と「△ヶ月保有する戦術期間」は独立した変数として設計する。

パラメータ	仮設定値	意味	探索範囲
TARGET_MONTHS（○ヶ月）	12ヶ月	資産増加の目標期間	6〜24ヶ月
HOLD_MONTHS（△ヶ月）	1・2・3ヶ月（並列）	各取引の保有単位	0.5〜6ヶ月

設計方針	モデルは各銘柄・各保有期間で①〜④の期待リターンを同時推論し、最高スコアの手段と期間の組み合わせを採用するマルチタスク出力構造とする。

## 2. 3つの主要検討事項

### 検討事項① ─ 再現率（Recall）の設計
「見逃し（偽陰性）コスト ＞ 無駄なシグナルコスト」という株式トレードの非対称性を踏まえ、再現率を最優先とした評価設計を行う。

指標	仮設定値	根拠・補足
再現率目標（RECALL_TARGET）	≥90%	機会損失を最小化する出発点。探索範囲：80〜95%
損失関数	Focal Loss（γ=2）	偽陰性を2倍ペナルティ。クラス不均衡に対応
評価指標	F-β（β=2）	Recallを2倍重視。精度低下を許容する非対称指標

⚠ 注意	再現率90%は偽陽性（不要シグナル）の増大を伴う。検討事項②の特徴数フィルタと2段階構成にすることで実用精度を確保する。

### 検討事項② ─ 特徴数条件（母数フィルタ）& ストップ高安除外ルール

特徴カウント設計
パラメータ	仮設定値	根拠
評価ウィンドウ（FEATURE_WINDOW）	20営業日（≈1ヶ月）	月次サイクルと整合する自然単位。探索：10〜60日
最低シグナル数（FEATURE_COUNT_MIN）	5個 / 窓内	母数20の25%。投票系モデルの経験的多数決閾値
特徴の母数（FEATURE_TOTAL）	20個（初期定義）	テクニカル10 + ファンダメンタル5 + 市場構造5

ストップ高安翌日 除外ルール（必須処理）
始値で約定できないケースをモデルが学習しないよう、以下3段階の除外パイプラインを実装する。

ステップ	処理内容	対象
A. フラグ付与	ストップ高安翌日に is_stop_limit_next_day = True を立てる	リアルタイムデータ
B. 仮算入→遡及除外	カウントに仮算入し、約定不可確認後に後処理で遡及除外	シグナル集計処理
C. 学習データから除去	訓練セット構築時に対象日の目的変数・入力特徴を両方除外	モデル学習フェーズ

### 検討事項③ ─ 統計学的フレームワーク（全体重視）
株式データはファットテール（裾の重い分布）であり正規分布の仮定が成立しにくい。3層の統計フレームワークで対応する。

σ閾値の活用
閾値	理論確率	用途・位置付け
±1σ	68.3%	通常変動域（参照値）
±2σ	95.4%	注意シグナル
±3σ	99.73%（≈0.27%超）	強シグナル判定（SIGMA_THRESHOLD 仮設定値）
±4σ	99.994%	極値・異常フラグ

3層統計フレームワーク構成
層	手法	目的・特徴
第1層（絶対シグナル）	3σ閾値 + Winsorization（クリップ3σ）	強制フラグ付与。極値でモデルを壊さない
第2層（分布適応）	Rolling Z-score（60日・125日窓）	ボラティリティ変化に動的対応。MADスコア併用
第3層（クロスセクション）	全銘柄の日次横断標準化 + ランク変換	セクター・時価総額バイアスを除去

パラメータ	仮設定値	根拠	更新トリガー
SIGMA_THRESHOLD	3.0σ	統計慣例 + 金融実務標準	3σ的中率が2.5σに劣った時
ROLLING_WINDOW_SHORT	60営業日（≈3ヶ月）	四半期ボラティリティ追従の最短窓	季節性銘柄でシグナル品質低下時
ROLLING_WINDOW_LONG	125営業日（≈6ヶ月）	上下半期サイクル1周分	長短Z-scoreが冗長化した時
WINSORIZE_CLIP	3.0σ	SIGMA_THRESHOLDと統一。情報保持しつつ勾配爆発防止	学習損失に爆発・消失兆候が出た時

## 3. 設計哲学 ─「仮設定→柔軟変更」の正当性

核心	最適化はデータがあって初めて機能する。動かす前の最適化は、過去の偏ったサンプルへの過学習（過剰適合）を生むだけで未来に使えない。

原則	内容	実装方法
① 根拠のある出発点	直感ではなく統計的・実務的先行知識から仮設定を導く	各パラメータに根拠コメントを明記
② 更新トリガーの事前定義	「何が起きたら何を変えるか」をあらかじめ決めておく	更新トリガー列をconfig.yamlに記述
③ config.yamlへの集約	コード内に数値を直書きせず設定ファイル一箇所に集める	全パラメータをyaml管理・git追跡

仮設定→更新フロー

ステップ	作業内容
Step 1	仮設定値でパイプラインを実装
Step 2	Walk-Forward Validationでバックテスト実行
Step 3	更新トリガー条件に抵触していないか確認
Step 4	根拠を明示してパラメータを更新
Step 5	再バックテストで改善を検証

## 4. モデルアーキテクチャ

推奨	Temporal Fusion Transformer（TFT）をベースとしたマルチタスク時系列深層学習モデル。多変量入力・静的メタデータ・既知将来変数を同時処理し、特徴重要度も解釈可能。

### 4.1　入力特徴群
カテゴリ	特徴例	初期数
価格系	OHLCV・リターン・ボラティリティ・ギャップ	〜5個
テクニカル系	MA・RSI・MACD・ボリンジャーバンド・ATR・OBV	〜10個
ファンダメンタル系	PER・PBR・ROE・営業CF・売上成長率	〜5個
市場構造系	日経・TOPIX・VIX・出来高比・信用倍率	〜5個
イベント系	決算日・配当落ち・ストップ高安フラグ	別途フラグ

### 4.2　出力・損失設計
要素	設計内容
出力スコア	4取引手段 × 4保有期間（1/2/3/6ヶ月）= 16スコア + 再現率調整付き分類ヘッド
損失関数	Focal Loss（Recall重視）＋ Sharpe比正則化項
評価指標	F-β（β=2）・累積超過リターン・最大ドローダウン
確率較正	Platt Scalingで確率出力をキャリブレーション

## 5. 学習・バックテスト設計

設計項目	方針
時系列分割	Walk-Forward Validation（時系列データ漏れなし）
除外データ	ストップ高安翌日・流動性極端低日を訓練・評価両方から除外
コスト組込み	手数料・スリッページ・信用コストを明示的にシミュレート
バイアス排除	ルックアヘッドバイアス・サバイバーシップバイアスを厳格に防止
特徴重要度	SHAPによる寄与度分析で冗長特徴を削除・追加（SHAP後に再調整）

## 6. 実装フェーズ計画

フェーズ	名称	主要タスク	完了条件
Phase 1	データ収集・前処理	全銘柄OHLCVデータ収集・ストップ高安フラグ実装・欠損値処理	除外パイプライン動作確認
Phase 2	特徴エンジニアリング	テクニカル指標計算・3層統計標準化（3σ・MAD）・クロスセクション正規化	全20特徴量の生成確認
Phase 3	ベースライン構築	シンプルモデルでRecall≥90%チューニング・F-β(β=2)での評価確立	Recall≥90%かつF-βベースライン計測
Phase 4	TFT本番モデル	Temporal Fusion Transformer実装・特徴数フィルタ統合・マルチタスク出力	16スコア出力の動作確認
Phase 5	バックテスト・本番化	Walk-Forward Validation実行・リスク検証・コスト込みシミュレーション・本番環境構築	Sharpe比・最大ドローダウン基準クリア

## 7. 設定ファイル（config.yaml）全体像
全パラメータを一元管理し、変更は1行で完結する。仮設定値と更新トリガーをコメントで明記。

```yaml
# config.yaml — 深層学習株式モデル 仮設定

period:
  target_months: 12      # 仮設定 | 範囲: 6–24
  hold_months: [1, 2, 3] # 仮設定 | 並列推論

signal:
  recall_target: 0.90    # 仮設定 | 範囲: 0.80–0.95
  feature_window: 20     # 仮設定 | 単位: 営業日
  feature_count_min: 5   # 仮設定 | 母数の25%
  feature_total: 20      # 仮設定 | SHAP後に再調整

stats:
  sigma_threshold: 3.0   # 仮設定 | 強シグナル境界
  rolling_short: 60      # 仮設定 | 単位: 営業日
  rolling_long: 125      # 仮設定 | 単位: 営業日
  winsorize_clip: 3.0    # 仮設定 | sigmaと揃える

exclude:
  stop_limit_next_day: true  # 固定 | ストップ高安翌日除外
```

## 8. リスク・留意事項

リスク項目	内容	対策
過学習	特定期間・銘柄への過剰適合	Walk-Forward Validation + SHAP定期確認
ファットテール	3σ超の実頻度が理論値より高い	MADスコア・Winsorization併用
流動性リスク	小型株は約定できない場合がある	出来高フィルタで除外・スリッページ加算
ストップ高安除外漏れ	学習データへの混入	is_stop_limit_next_day フラグの3段階チェック
後付け最適化	結果を見て恣意的にパラメータを変更	更新トリガー条件を事前定義し遵守

本計画書は仮設定値を含む。バックテスト結果に基づき根拠を明示して随時更新すること。
